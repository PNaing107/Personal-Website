Week 2: Portfolio Project

This week was our first project week. It was time to put the theory we learnt last week about HTML and CSS into practice by building our own portfolio website. The brief was pretty straightforward. We had to design and build a responsive website with at least 3 sections: an About Me, Portfolio and Contact section. We were also asked to build this on Replit which is a browser based IDE (Integrated Development Environment).

Because this would be our personal website I felt great pressure to make the website look as impressive as I could and spent far too long looking online at other peopleâ€™s work for inspiration. This can be quite dangerous in my opinion. The examples that come up on top of the Google searches are usually ones made by seasoned pros who are using far more advanced languages than just HTML and plain CSS. This can feel demoralising for us mere mortals so I had to remind myself to stop procrastinating as this portfolio can always be updated in the future. With that in my mind I quickly came up with some simple mobile and desktop wireframes for my website. I decided to go with a fixed header and footer as my website didnâ€™t need a navigation bar and I wanted to give it a native app feeling on mobile devices. This was approved by my trainer Neal so then I started writing out the HTML and then styling it with CSS. Having a wireframe, essentially a blueprint for my website, is so important as it made writing the HTML and CSS a breeze.
